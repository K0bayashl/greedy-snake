# 代码审查技能

审查项目代码质量，提供改进建议。

## 审查范围

1. **最新改动** - 检查最近修改的文件
2. **代码质量** - 可读性、可维护性
3. **潜在问题** - Bug、安全漏洞、性能问题
4. **代码规范** - 命名、注释、结构

## 执行步骤

1. 运行 `git status` 查看改动文件
2. 读取改动的文件内容
3. 分析代码质量
4. 输出审查报告

## 输出格式

```
# 代码审查报告

## 改动文件
- [文件列表]

## 优点
- [列出做得好的地方]

## 问题
| 文件 | 行号 | 严重程度 | 问题描述 | 建议 |
|------|------|----------|----------|------|

## 改进建议
[总体建议]
```

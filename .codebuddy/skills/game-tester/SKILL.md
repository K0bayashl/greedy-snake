# 游戏测试技能

你是一位专注于游戏功能测试的专家。

## 执行策略

### 自己完成（简单场景）
- ✅ 单个功能点测试
- ✅ 验证明确的问题修复
- ✅ 快速冒烟测试
- ✅ 测试范围明确

**操作**：自己阅读代码、运行游戏、输出报告

### 调用 tester SubAgent（复杂场景）
- ❌ 全面回归测试（多个功能）
- ❌ 需要深度测试用例设计
- ❌ 复杂 Bug 的复现和定位
- ❌ 需要编写自动化测试脚本

**操作**：使用 `Task(subagent_type="tester", prompt="...")`

## 测试流程

1. **评估测试范围** - 判断是自己做还是调用 SubAgent
2. **执行测试** - 按策略选择方式
3. **报告结果** - 输出详细的测试报告

## 测试类型

| 类型 | 说明 |
|------|------|
| 功能测试 | 验证功能是否按预期工作 |
| 边界测试 | 测试极端值和边界条件 |
| 回归测试 | 确保修改未影响现有功能 |
| 用户场景 | 模拟真实用户操作流程 |

## 测试清单

### 基础功能
- [ ] 游戏启动正常
- [ ] 方向控制响应正确
- [ ] 吃食物增长和得分
- [ ] 碰撞检测正确
- [ ] 暂停/恢复功能

### 新功能
- [ ] 按需求规格验证每个新功能点
- [ ] 测试边界条件
- [ ] 测试异常情况处理
